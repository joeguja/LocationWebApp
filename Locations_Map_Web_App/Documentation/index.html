XML

The locations.xml file located in the root of the Application folder is the source of the data for the application.  The layout below is a sample entry.

<?xml version="1.0" encoding="UTF-8"?>

<locations>

  <location>

    <title>Hollywood Sign</title>

    <address>3204 Canyon Lake Drive, Los Angeles, CA</address>

    <details>The Hollywood Sign is a famous landmark in the Hollywood Hills area of Mount Lee, Santa Monica Mountains, in Los Angeles, California. The iconic sign spells out the name of the area in 45-foot-tall (14 m) and 350-foot-long (110 m) white letters.</details>

    <images>

      <image>1_1.jpg</image>

      <image>1_2.jpg</image>

    </images>

    <marker>/css/img/markers/photography.png</marker>

    <lat>34.134103</lat>

    <lng>-118.321694</lng>

  </location>

</locations>

 

In the js/config.js file you can see the following entry:

var locationFields = [

        {name: 'title', type: 'string'},

        {name: 'address', type: 'string'},

        {name: 'details', type: 'string'},

        {name: 'images', type: 'object'},

        {name: 'thumb', type: 'string'},

        {name: 'marker', type: 'string'},

        {name: 'lat',       type: 'float'},

        {name: 'lng',   type: 'float'}

    ]

This tells the application what data to store when initializing.  It is populated by default with the minimal fields you need to have in order for the app to function but you can easily add more.

Here is a list of the default fields:

  • title: the title of the entry
  • address: the address of the entry
  • details: a description of the location
  • images: this is a list of images attributed to the location to be displayed in the image carousel
  • marker: this is the icon that will appear on the map for this location
  • lat: the latitude of the location
  • lng: the longitude of the location

Adding an additional field

The first step in adding an additional field is creating a tag for it in the XML file, for this example we will add a year entry since the default locations are monuments to bedisplayed in the details section.

in the locations.xml file we add the following line with the <year> tag somewhere in between the <location></location> tag:

<title>Hollywood Sign</title>
<year>1923</year>

Next, we need to edit the config.js file to tell the application about our new field.

var locationFields = [
... 
        {name: 'lng',   type: 'float'},
        {name: 'year', type: 'string'},
...
    ]

The last step is to tell the application were to display the new data in the details tab, goto the last line in the config.js file and edit it like so:

var detailsTemplate =  '<div><tpl if="year"><p>Built in {year}</p></tpl><tpl if="address"><address>{address}</address></tpl><tpl if="details"><p>{details}</p></tpl></div>';

Notice we added <tpl if="year"><p>Built in {year}</p></tpl> This will output "Built in 1923 at the top of the panel now if data exists for that entry.

PHP

The locations app comes with one PHP file: json.php.  This is a quick and simple way to convert the data in the XML file to JSON which the app can interpret.  If you plan on using data from a database for your implementation you will need to create your own PHP file that connects to your database and then JSON encode your results.  If your server doesnt have PHP capabilities, you can always encode the data yourself, there are several tools online like this one that will allow you to create a .json file, just change the JSON_SOURCE variable in the config.js file and the application will use your data.

<?php

//Check if the locations.xml file exists, if it does encode it to JSON for the app

if (file_exists('locations.xml')) {

     $xml = simplexml_load_file('locations.xml');

     $json = json_encode($xml);

     print_r($json);

} else {

     exit('Failed to open locations.xml.');

}

?>

The remaining javascript files fall into these categories:

  • Models: There is one model file, a model communicates with the data, in the case of this app, the location information.
  • Views: These files display the data and create the different panels of the application
    • js/app/views/Viewport.js - The container view for the whole app
    • js/app/views/home/HomeIndexView.js - The homepage
    • js/app/views/map/MapMain.js - The main map on the map tab
    • js/app/views/map/MapDetail.js - The detail map for locations opened on the main map page
    • js/app/views/location/Locations.js - The list of locations
    • js/app/views/location/LocationDetail.js - The detail panel for each location
    • js/app/views/location/LocationInfo.js - The text panel for the detail
    • js/app/views/location/LocationImage.js - The image carousel in the detail panel
  • Controllers: These files interact with the data from the model
    • js/app/controllers/MapController.js - Contains actions for adding markers, updating location, opening detail panels

CSS and Images

There are two CSS files that can be found in the application directory.  The first application.css, is the bulk of the styling for the app.  This file is created dynamically using sass and compass and may be too advanced for some.  For this reason you will find a file by the name of "custom.css" which will allow you to override any styles you would like.

Several map marker icons have also been included in the folder css/img/markers.  Browse through these png files as you can include a different icon for each location entry and add as many custom images as you would like in png format.

Using Compass to edit theme (advanced users)

Once you have installed Compass, navigate to Compass/scss and open the application.scss file.  When compass compiles the .scss file it uses base colors to create the gradient you see in the UI of the app.  You can easily change the entire color scheme by simply caging the value of $base-color.  For a more detailed list of te variables you can edit, view the sencha touch introduction on themeing.

Additional info

If you plan on using this applications as a mobile alternative you can use the following code to redirect iPhone and iPad to the location of the app on your server, in this example the app folder was placed in a folder called "mobile_directory":

<script type="text/javascript"> 
	if ((navigator.userAgent.indexOf('iPhone') != -1) || (navigator.userAgent.indexOf('iPod') != -1) || (navigator.userAgent.indexOf('iPad') != -1)) {
	document.location = "http://your.com/mobile_directory";
	}
</script>
